# Analysis Rules

Analyze sources cautiously and connect findings to the configured topic and current user.

## Order

1. Identify main evidence.
2. Summarize the key point from each main source.
3. Compare agreements across sources.
4. Note differences or conflicts.
5. Assess report confidence.
6. Explain practical relevance to the current user/topic.
7. Suggest 3 small actions for today, when useful.

## Claim Strength

Strong claim:

- supported by fetched high-confidence evidence
- clear source content
- no major conflict

Medium claim:

- supported by medium evidence or multiple sources
- plausible but not fully verified

Weak claim:

- only social signal, snippet, fetch failure, or low-confidence source

Write weak claims cautiously. Do not present them as facts.

## Cross-check

Check:

- whether sources describe the same event
- which source is original
- which sources only repeat others
- conflicts in dates, numbers, claims, or conclusions
- whether official/vendor confirmation exists

If sources conflict, explain the conflict and prefer official/vendor/security research sources.

## Practical Relevance

Relate the analysis to the configured topic and the current user's likely use of the report.

Avoid generic statements such as:

- "this is important"
- "you should follow the trend"
- "the market is moving fast"

Be specific and actionable.

## Actions

If action items are included, provide exactly 3 small actions:

1. Action
   - Reason:
   - Expected result:

Keep actions realistic for the same day.

## Do Not

- invent data, numbers, URLs, or conclusions
- use fetch-failed sources as conclusions
- claim direct Facebook/TikTok crawling
- overstate weak evidence
- use hype words unless sources support them
